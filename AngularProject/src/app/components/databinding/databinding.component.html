DATABINDING

In Angular, databinding is the act of passing data from the backend (our typescript file) to the view or vice versa. There are 3 ways to perform one-way data binding in Angular. That is, data is either passed from the backend to the view or from the view to the backend. When data is simultaneously passed both forward and backward, this is called two-way databinding.

STRING INTERPOLATION

In order to perform string interpolation, you need to write the item to be evaluated in between double curly braces like so. It is essentially a block typescript as innerHTML. You cannot perform interpolation with element attributes. Interpolation must be performed in between tags, not in them.

Property Binding

Property binding is a one-way databinding technique that binds from the back to the front, but this time we apply it WITHIN element tags. We do this by surrounding what would be the "attribute" of the element with square brackets [likethis].

Here I am!

EVENT BINDING

Event binding is the third approach to one-way databinding in which we take input, or an event from the frontend, and transfer it to the backend (so our typescript portion of our component). This is done within the element's tag, except this time we use parentheses instead of brackets. We will typically wrap these around events.

TWO-WAY DATABINDING

We can set up two-way databinding, where data we input is fed back to a component and can be brought back all in the same action. In order to do this, we need to invoke both property binding AND event binding alongside a directive called ngModel.
